# Preface: A New Start

## Important Points

- Supervised practical sessions run in **Weeks 2-11** of Semester 2.
- **Week 1 is self-study** for lab preparation: review introductory material and watch briefing video content.
- Lab access is usually available during revision and assessment periods for unsupervised catch-up/improvement time.
- Knowledge from practicals supports selected questions in the final exam and resit.
- Practical work includes C programming for algorithm development and CPU pipeline problem solving.
- Exam questions related to practical content **will not require writing C code**.

## Delivery Plan

- Practical sheets are released every Friday in the week before the intended lab session.
- Most tasks are self-contained one-week activities; some continue across multiple weeks.
- Work combines:
  - investigating pipeline behavior and concepts
  - implementing algorithms/code for CPU execution pipeline problems

The schedule below is the current expected delivery pattern (subject to occasional in-term updates):

## Solutions

Solutions may be released at certain points, typically after the scheduled practical study weeks for a task have ended.

Practical sessions are designed as experimental learning sessions. The goal is not only to "get the right answer", but to build deeper understanding of pipeline behavior and code-efficiency trade-offs through exploration and reasoning.

Published solutions are helpful references, but they are not a substitute for your own investigation and problem solving during labs.

If you fall behind, a useful strategy is:

1. Review released solutions before the next week's session.
2. Use unsupervised lab time to run and inspect the solutions.
3. Re-read that week's lab sheet and map each step to the implementation.
4. Bring focused questions to tutors/GTAs in the next session.



## CCCP (Crash Course in C Programming)

The CCCP primer was incorporated into the SYS2 VLE content in Semester 1. You should already have access via the VLE and can revisit it to refresh C programming knowledge.

Many concepts can also be reinforced during labs with tutor/GTA support and peer collaboration.

## Can I Supplement Practical Study Off-site?

You should attend practical sessions in person where possible, as interaction and lab support are important for skill development.

To supplement study off-campus, you may be able to:

- install Ubuntu (or run an Ubuntu VM)
- install `gcc` / `g++`
- run the same command-line workflow used in labs

Notes:

- Formal support for personal setups is not provided.
- Tutors may still offer limited general advice.
- Practicals were developed on Ubuntu 22.04.1 LTS (VMWare), which is known to work.
- Other Linux distributions may also work but are not formally tested.
- You may also be able to access lab machines remotely for out-of-hours work.

## Can I Use Extra Software Tools in Labs?

The practical design assumes minimal tools: a text editor plus command-line scripts.

You may use an IDE if you prefer, but:

- tutors/GTAs may not be familiar with all IDE-specific features
- command-line debugging remains a useful skill to develop in these sessions
