# Lab 5: Structural Hazards (Memory)

## Overview

**Structural hazards** arise when the machine cannot serve all memory accesses at once, for example because of limited memory chips, buses, or read and write ports. The details depend on the processor and system design.

This lab looks at **memory-related structural conflicts** (contention for memory interfaces), not register data hazards.

## This week

- Investigate how memory-side structural limitations show up in the pipeline model.
- Experiment with scheduling or structural fixes that reduce conflicts.
- Relate your observations to the lab sheet tasks.
